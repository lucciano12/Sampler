Documentación de las APIs externas de música tal como las consume el proyecto Sampler desde Angular vía HttpClient. ⚠️ Este archivo documenta APIs de terceros, no endpoints propios (el proyecto es frontend puro, sin backend). Cada endpoint describe exactamente los campos que el servicio correspondiente lee de la respuesta. Flujo de enriquecimiento: Discogs (búsqueda base, portada, estilo) → TheAudioDB (BPM, género, mood, portada, YouTube, MBID) → AcousticBrainz (BPM preciso, key — usa el MBID de TheAudioDB)

Discogs

Búsqueda de releases musicales (portada, estilo, género)

Buscar releases por estilo o por artista + título

Motor de búsqueda principal del proyecto. Se consume desde DiscogsService en dos modos: búsqueda por estilo (buscarPorEstilo) que obtiene hasta 25 releases de un estilo dado para poblar el catálogo, y enriquecimiento (enrichFromDiscogs) que busca un release específico por artista + título para obtener portada y estilo.

Authorizations:
DiscogsToken

Tipo de entidad a buscar (el proyecto siempre usa release)

integer

Example: per_page=25

Cantidad de resultados por página (el proyecto usa 25)

TheAudioDB

Metadatos de tracks (BPM, género, mood, YouTube, MBID)

Buscar track por artista y título

Enriquece los metadatos del sampler con BPM, género, mood, portada thumbnail, link de YouTube y MusicBrainz ID (usado luego por AcousticBrainz). El servicio enrichFromAudioDb parsea intBPM a número y lo asigna como tempo del sampler. También asigna strGenre como genero.

query Parameters
s
required
string
Example: s=Stevie Wonder

Nombre del artista

t

string
Example: t=Superstition

Título del track

AcousticBrainz

Análisis de audio (BPM preciso, clave/tonalidad)

Obtener datos de audio de bajo nivel por MBID

Devuelve BPM (detectado por análisis de audio) y clave/tonalidad musical para un recording identificado por su MusicBrainz ID (MBID). El MBID se obtiene previamente desde TheAudioDB (strMusicBrainzID) o desde MusicBrainz como fallback. El servicio enrichFromAcousticBrainz redondea el BPM a entero y combina key_key + key_scale como key del sampler.

path Parameters
mbid
required
string <uuid>
Example: a1b2c3d4-e5f6-7890-abcd-ef1234567890

MusicBrainz ID (UUID) del recording

Responses

TheAudioDB API — enriquecimiento de metadatos

AcousticBrainz API — datos de audio de bajo nivel

https://acousticbrainz.org/api/v1/{mbid}/low-level
